Project Codename: Zenith

A high-performance asynchronous data pipeline built on modern Python and Bulma.

Status: Deployed Stack: Flask & PostgreSQL Last Update: Oct 2024

Project Overview

This system was developed to solve the challenge of processing millions of concurrent financial transactions. The architecture is deliberately decoupled, emphasizing microservices principles to ensure scalability and fault tolerance. Our primary design goal was achieving a **P99 latency below 10ms**.

Key Objectives & Modules

Latency

Ensure all core API endpoints respond in under 50ms, leveraging aggressive caching.

Architecture

Implement the Command Query Responsibility Segregation (CQRS) pattern.

Learning

Master Docker Compose for reproducible, environment-agnostic deployment.

Data Flow and Services

The system comprises three main services: an ingestion worker (Celery), a state management service (Redis), and a reporting API (Flask). Data flows uni-directionally to prevent deadlocks.
